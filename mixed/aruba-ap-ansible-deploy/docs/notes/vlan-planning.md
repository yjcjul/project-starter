## 🔧 技術考量

### 1. 為什麼每台 AP 要獨立 VLAN？

| 原因 | 說明 |
|------|------|
| 流量隔離 | 避免廣播風暴影響整體網路 |
| 故障隔離 | 單一 AP 問題不影響其他 AP |
| 安全性 | 限制橫向移動攻擊 |
| QoS 控制 | 更精細的頻寬管理 |

### 2. 管理 VLAN vs 用戶 VLAN

```
┌────────────────────────────────────────────────────────────┐
│                        AP 配置                             │
├────────────────────────────────────────────────────────────┤
│                                                            │
│   管理 VLAN (Native VLAN)     用戶 VLAN (Trunk)           │
│   ┌─────────────┐            ┌─────────────┐              │
│   │ VLAN 101    │            │ VLAN 24     │  教職員      │
│   │ AP 管理流量 │            │ VLAN 25     │  學生        │
│   │ Controller  │            │ VLAN 26     │  訪客        │
│   │ 通訊        │            └─────────────┘              │
│   └─────────────┘                                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 3. IP 位址規劃

假設每個 AP VLAN 使用 /30（4 個 IP，2 可用）：

```
VLAN 101: 10.1.101.0/30
  - 10.1.101.1 → Gateway
  - 10.1.101.2 → AP-001

VLAN 102: 10.1.102.0/30
  - 10.1.102.1 → Gateway
  - 10.1.102.2 → AP-002
```

或使用 /28（16 個 IP）以便未來擴展：

```
VLAN 101: 10.1.101.0/28
  - 10.1.101.1 → Gateway
  - 10.1.101.2-14 → 可用
```
